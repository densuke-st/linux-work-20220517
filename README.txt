各ディレクトリにあるREADME.txtに問題があるので、そこで作業してください。

例えば問題001なら
$ cd 001
$ ls  # README.txtがあるのを確認
$ cat README.txt # 多分画面1つを超えるような量はない

README.txtの内容に従い作成してください。
問題内容のチェックについては、このディレクトリにあるcheck.shを使って下さい。

例1: 002のディレクトリにいるとき
$ ../check.sh # "."2つ

例2: トップのディレクトリ(このファイルのあるディレクトリ)にいるとき
$ ./check.sh # "."1つ

作業状態を全て消すときは、`cleanup.sh` を使うと一掃できます。
ただし正解のところも消えます。実行はトップのディレクトリ(このファイルのあるディレクトリ)で行って下さい。
$ ./cleanup.sh
